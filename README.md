#Сверхстабильный двухколесный балансирующий робот

> **«Лучший робот, созданный гениями человечества, великолепно балансирующий с помощью идеально написанного ПО для акселерометра».**

Репозиторий содержит программное обеспечение и математическую модель для автономной системы динамической балансировки двухколесного робота на базе микроконтроллера Arduino и MEMS-датчика MPU-6050 (акселерометр-гироскоп).

---



## 📐 Физико-математическая модель

Для удержания баланса используется уравнение движения, основанное на втором законе Ньютона для вращательного движения относительно оси колес:

$$I\ddot{\theta} = mgh \sin\theta - M_{\text{motor}}$$

Где:
* $I$ — момент инерции точечной массы
* $\ddot{\theta}$ — угловое ускорение системы
* $mgh \sin\theta$ — опрокидывающий момент силы тяжести
* $M_{\text{motor}}$ — компенсирующий момент, создаваемый двигателями

### Расчет критического угла
Максимальный угол наклона, который робот способен динамически скомпенсировать при заданной мощности моторов, рассчитывается по формуле:

$$\theta_{\text{max}}^{\text{(dyn)}} = \arcsin\left(\frac{M_{\text{max}}}{mgh}\right)$$

**Параметры нашей системы:**
* $g = 9.81 \, \text{м/с}^2$ — ускорение свободного падения
* $m = 0.8 \, \text{кг}$ — масса робота (800 грамм)
* $h = 0.13 \, \text{м}$ — высота центра масс (13 см)
* $M_{\text{max}} = 0.1 \, \text{Н}\cdot\text{м}$ — максимальный крутящий момент двигателей

$$\theta_{\text{max}}^{\text{(dyn)}} = \arcsin\left(\frac{0.1}{9.81 \cdot 0.13 \cdot 0.8}\right) \approx 5.6^{\circ}$$

Если робот отклонится более чем на **5.6°**, крутящего момента моторов не хватит для возврата в вертикальное положение, и произойдет падение.

##  Настройка ПИД-регулятора

Стабилизация осуществляется посредством программного ПИД-регулятора. Текущие оптимальные коэффициенты:

| Параметр | Значение | Описание | Физический смысл |
| :--- | :---: | :--- | :--- |
| **Пропорциональный ($K_p$)** | `25.0` | Сила возврата | Определяет силу противодействия наклону. Избыток вызывает вибрацию. |
| **Интегральный ($K_i$)** | `0.40` | Устранение статической ошибки | Компенсирует постоянный наклон и уход с точки из-за дисбаланса. Избыток вызывает раскачку. |
| **Дифференциальный ($K_d$)**| `1.2` | Демпфирование | Гасит колебания. Избыточное значение приводит к резким рывкам двигателей. |
| **Смещение нуля (`targetAngle`)** | `0.5°` | Нулевая точка | Компенсация физического дисбаланса конструкции (смещение центра масс). |
| **Лимит ШИМ (`MaxSpeed`)** | `250` | Ограничение мощности | Аппаратное ограничение максимального заполнения ШИМ-сигнала. |

##  Структура репозитория

* `main.ino` — основной рабочий код для платы Arduino. Реализует опрос датчика MPU-6050 по протоколу I2C, фильтрацию данных, расчет ПИД-алгоритма и генерацию ШИМ для драйверов моторов.
* `README.md` — документация проекта.

---

##  Быстрый запуск

1. Подключите акселерометр MPU-6050 к Arduino (SDA -> A4, SCL -> A5 для платы Uno/Nano).
2. Откройте файл `main.ino` в Arduino IDE.
3. Установите необходимые библиотеки для работы с I2C и MPU-6050 (если используются).
4. Загрузите скетч в микроконтроллер.
5. Поставьте робота на ровную поверхность и включите питание.
